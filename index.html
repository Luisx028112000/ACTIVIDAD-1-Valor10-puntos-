Tema 1: Elementos de Páginas Web

1. Web 2.0:

La Web 2.0 se refiere a la evolución de la Web que permitió una mayor interacción entre usuarios y contenido. Este cambio significó pasar de una Web estática a una más dinámica y colaborativa.

Características principales de la Web 2.0:

• Interactividad y colaboración: Los usuarios pueden crear, compartir y editar contenido. Ejemplos incluyen blogs, wikis y redes sociales.

• Aplicaciones Web: Uso de aplicaciones que se ejecutan directamente en el navegador, como Google Docs.

• Tecnologías clave: AJAX, JavaScript, HTML5, CSS3. • Redes sociales y comunidades en línea: Plataformas como Facebook, Twitter y YouTube.

2. Web 3.0

La Web 3.0, también conocida como la Web Semántica, busca mejorar la forma en que se organiza y se accede a la información mediante el uso de tecnologías avanzadas.

Características principales de la Web 3.0:

• Datos conectados: Uso de tecnologías como RDF (Resource Description Framework) y OWL (Web Ontology Language) para conectar datos de manera significativa.

• Inteligencia artificial: Utilización de AI para mejorar la búsqueda y el procesamiento de información.

• Personalización: Servicios más personalizados basados en el comportamiento y las preferencias del usuario.

•Blockchain y descentralización: Introducción de tecnologías descentralizadas que mejoran la seguridad y la privacidad de los usuarios.

3. La W3C y los estándares

La W3C (World Wide Web Consortium) es una organización internacional que desarrolla estándares para asegurar el crecimiento a largo plazo de la Web.

Roles y contribuciones de la W3C:

Establecimiento de estándares: Desarrolla y mantiene estándares abiertos para garantizar la interoperabilidad y el acceso universal.

• Tecnologías fundamentales: HTML, CSS, XML, y otros estándares que forman la base de la Web.

• Accesibilidad: Promueve la accesibilidad web a través de directrices como las WCAG (Web Content Accessibility Guidelines).

• Seguridad y privacidad: Trabaja en estándares que protegen la seguridad y la privacidad de los usuarios.

EStos componentes han sido cruciales para el desarrollo y la evolución de la Web, permitiendo que se convierta en una herramienta poderosa y omnipresente en la vida moderna.

II Lenguaje de etiquetas

1. SGML (Standard Generalized Markup Language)

SGML es un estándar para definir lenguajes de marcado utilizados en la creación de documentos estructurados. Desarrollado en la década de 1980, SGML proporciona un marco para definir la estructura y el formato de documentos textuales.

Características principales:

Flexibilidad: Permite definir diferentes tipos de documentos y sus estructuras.

Complejidad: Es complejo y puede ser difícil de implementar completamente.

Usos: Fue ampliamente utilizado en la industria de la publicación y en entornos donde se necesitaba una estructura de documentos robusta y personalizable.

2. XML (Extensible Markup Language)

Es un lenguaje de marcado derivado de SGML, diseñado para ser más simple y más flexible. Fue desarrollado por el W3C y es ampliamente utilizado para la transferencia de datos entre sistemas.

Características principales:

Simplicidad: Más fácil de usar y comprender en comparación con SGML.

Los usuarios pueden definir sus propias etiquetas.

3. HTML (HyperText Markup Language)

HTML es el lenguaje estándar utilizado para crear y diseñar páginas web. Define la estructura básica de un documento web y es interpretado por navegadores web.

Características principales:

• Estructura básica: Define la estructura de una página web utilizando elementos como , , , etc.

• Hipertexto: Permite la creación de enlaces entre documentos.

• Multimedia: Soporta la integración de imágenes, videos y otros tipos de contenido

4. XHTML (Extensible HyperText Markup Language)

XHTML es una versión más estricta y basada en XML de HTML. Fue desarrollado para combinar las capacidades de HTML con la rigidez y extensibilidad de XML.

Características principales:

• Estricto: Requiere que los documentos sean bien formados y sigan una sintaxis más estricta.

• Compatibilidad: Facilita la compatibilidad entre diferentes sistemas y plataformas.

• Transición: Fue diseñado como un paso intermedio entre HTML y XML, aunque no tuvo una adopción tan amplia como HTML5..

5. HTML5

Es la última versión del lenguaje HTML, diseñada para abordar las limitaciones de versiones anteriores y proporcionar nuevas funcionalidades para aplicaciones web modernas.

Características principales:

Nuevos elementos: Introducción de nuevos elementos semánticos como header, footer, article, section, etc.

Multimedia: Mejora el soporte para multimedia con elementos como audio> y video>.

APIs: Integración de nuevas APIs que permiten funcionalidades avanzadas como gráficos en 2D y 3D, almacenamiento local, y más

Compatibilidad: Diseñado para ser compatible con dispositivos móviles y adaptarse a diferentes tamaños de pantalla.

Estos lenguajes de etiquetas han sido fundamentales en el desarrollo de la web, cada uno aportando sus propias fortalezas y características que han permitido la evolución continua de las tecnologías web.

III. Otros lenguajes relacionados al desarrollo web

CSS3:Las Hojas de Estilo en Cascada (CSS) y su versión más reciente, CSS3, se utilizan para diseñar y maquetar páginas web. CSS3 introduce nuevas características como animaciones, transiciones y capacidades de diseño responsive.

JavaScript: Un lenguaje de scripting versátil que permite crear páginas web interactivas. Es una tecnología central de la web, junto con HTML y CSS, y se usa para crear contenido dinámico y mejorar la experiencia del usuario.

Ajax: Significa JavaScript y XML Asíncrono. Es un conjunto de técnicas de desarrollo web que permite a las páginas web actualizarse de manera asíncrona intercambiando pequeños fragmentos de datos con el servidor en segundo plano, mejorando la experiencia del usuario al cargar datos sin necesidad de refrescar la página.

Tecnologías y lenguajes del lado del servidor: Se refiere a las tecnologías y lenguajes del lado del servidor utilizados para construir la parte backend de las aplicaciones web. Ejemplos incluyen PHP, Python, Ruby, Node.js y Java, que manejan el procesamiento de datos, la lógica del servidor y las interacciones con bases de datos.

Navegadores: Son programas que renderizan y muestran el contenido web. Ejemplos incluyen Google Chrome, Mozilla Firefox, Safari y Microsoft Edge. Cada navegador tiene su propio motor de renderizado y puede manejar los estándares web de manera diferente.

Motores de renderizado: Son componentes de los navegadores web que muestran el contenido web. Ejemplos incluyen Blink (usado por Chrome), Gecko (usado por Firefox) y WebKit (usado por Safari).

Herramientas de desarrollo: Son aplicaciones de software utilizadas para crear, depurar y optimizar código. Esto incluye entornos de desarrollo integrados (IDEs), editores de código, sistemas de control de versiones y herramientas de depuración.

Software WYSIWYG: "What You See Is What You Get" (Lo Que Ves Es Lo Que Obtienes) se refiere a editores que permiten diseñar y maquetar páginas web visualmente sin necesidad de escribir código. Ejemplos incluyen Adobe Dreamweaver y Microsoft Expression Web. Estos programas muestran una vista previa del diseño final mientras trabajas.

• Otras alternativas:

Figma: Una herramienta de diseño colaborativo basada en la web, muy popular para la creación de interfaces de usuario (UI) y prototipos. Permite la colaboración en tiempo real y es ampliamente utilizada para el diseño de aplicaciones web y móviles.

InVision: Plataforma de diseño colaborativo y prototipado que permite a los equipos de diseño crear prototipos interactivos y obtener retroalimentación de manera eficiente. Se integra con otras herramientas de diseño y facilita la colaboración entre diseñadores y desarrolladores.

Zeplin: Una herramienta que facilita la colaboración entre diseñadores y desarrolladores. Permite a los diseñadores exportar sus diseños desde herramientas como Figma, Sketch o Adobe XD y proporciona especificaciones detalladas para los desarrolladores, como medidas y colores.

GitHub: Una plataforma de desarrollo basada en Git que permite a los desarrolladores gestionar el control de versiones de su código, colaborar en proyectos y compartir su trabajo con otros. GitHub es esencial para el trabajo en equipo en proyectos de software y proporciona herramientas para revisar y fusionar cambios en el código.

Inspección de código: Herramientas que permiten examinar y depurar el código fuente de una aplicación web o software. Los navegadores modernos como Google Chrome y Mozilla Firefox tienen herramientas de desarrollo integradas que permiten inspeccionar el HTML, CSS y JavaScript de una página web, analizar el rendimiento y depurar problemas en tiempo real.

I. Tipos de Páginas Web

Páginas Web Estáticas: Contienen contenido fijo que no cambia con frecuencia. Están diseñadas con HTML y CSS y son adecuadas para sitios que no requieren interacción del usuario ni actualizaciones frecuentes.

Páginas Web Dinámicas: Generan contenido en tiempo real basado en interacciones del usuario o en datos almacenados en una base de datos. Usan lenguajes del lado del servidor como PHP, Python o JavaScript junto con bases de datos.

1.Páginas Web Interactivas: Ofrecen una experiencia de usuario enriquecida mediante la interacción en tiempo real, como formularios interactivos, animaciones y elementos multimedia. Usan tecnologías como JavaScript y frameworks de front-end.

2.Páginas Web de Comercio Electrónico: Están diseñadas para permitir la compra y venta de productos o servicios en línea. Incluyen características como catálogos de productos, carritos de compras, y sistemas de pago.

3.Páginas Web de Portafolio: Se utilizan para mostrar trabajos o proyectos de un individuo o una empresa. Son comunes entre diseñadores, artistas y profesionales creativos para exhibir su trabajo y habilidades.

4.Páginas Web de Blogs: Se centran en la publicación de artículos o contenido regularmente actualizado. Suelen tener características como comentarios y suscripciones para mantener a los lectores comprometidos.

5.Páginas Web Corporativas: Representan a empresas o instituciones y proporcionan información sobre sus servicios, productos, historia, y contacto. Son importantes para establecer la presencia en línea de una organización.

6.Planificación y Definición de Objetivos: Antes de comenzar el diseño, es fundamental definir los objetivos de la página web, el público objetivo y los requisitos funcionales.

7.Estructura y Arquitectura de Información: Crear un mapa del sitio que organice la información de manera lógica y estructurada, facilitando la navegación del usuario.

8.Diseño y Prototipo: Diseñar la apariencia visual de la página web, incluyendo la elección de colores, tipografías y layout. Crear prototipos o wireframes para visualizar la disposición de los elementos.

9.Desarrollo: Codificar la página web utilizando HTML, CSS y JavaScript para el diseño y funcionalidad. Para páginas dinámicas, implementar tecnologías del lado del servidor y bases de datos.

10.Pruebas y Debugging: Realizar pruebas exhaustivas para asegurar que la página web funcione correctamente en diferentes navegadores y dispositivos. Identificar y corregir errores.

11.Optimización y SEO: Optimizar la página web para tiempos de carga rápidos y asegurarse de que sea amigable con los motores de búsqueda mediante técnicas de SEO (Search Engine Optimization).

12.Lanzamiento y Mantenimiento: Publicar la página web en un servidor y realizar un seguimiento continuo para mantenerla actualizada, segura y funcional.

13.Accesibilidad y Usabilidad: Asegurarse de que la página sea accesible para todos los usuarios, incluidas personas con discapacidades, y que sea fácil de usar.

Tema 3: Estructura De Una Página Web

I. Jerarquía de directorio de un proyecto web.

La jerarquía de directorio de un proyecto web organiza los archivos y recursos necesarios para desarrollar una página web de manera estructurada. Un esquema común incluye: /root: Carpeta principal del proyecto. /css: Contiene archivos de hojas de estilo en cascada (CSS). /js: Contiene archivos de JavaScript. /images: Contiene imágenes y gráficos utilizados en el sitio web. /fonts: Contiene fuentes personalizadas si se utilizan. /includes: Carpeta para archivos comunes que se incluyen en varias páginas, como encabezados y pies de página. /docs: Documentación del proyecto. index.html: Archivo principal de la página de inicio. contact.html: Ejemplo de archivo de una página de contacto. about.html: Ejemplo de archivo para una página sobre nosotros. Esta estructura puede variar según el tamaño y la complejidad del proyecto, pero mantiene una organización clara y accesible.
